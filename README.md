# 🚀 Real-Time Streaming Big Data Pipeline

## 📌 Overview

An end-to-end real-time streaming data pipeline that ingests user data from the Random User API, processes streaming data using distributed systems, stores the results for analytics, and visualizes insights in real time.

---

## 📖 Project Description

This project simulates a production-grade real-time streaming environment using modern Big Data technologies. User data is continuously fetched from the Random User API, streamed through Apache Kafka, processed using Apache Spark, and stored in Hadoop HDFS and InfluxDB for analytics and monitoring.

The pipeline demonstrates how real-time data can be collected, processed, stored, and visualized using a scalable Big Data architecture.

### Key Features

* Real-time data ingestion from Random User API
* Distributed stream processing with Apache Spark
* Scalable storage using Hadoop HDFS
* Time-series analytics with InfluxDB
* Real-time monitoring and visualization with Grafana
---

## 💻 Deployment

The project was deployed and tested on a Linux Virtual Machine (VM) in a real-time streaming environment.

---

## 📊 Results

* Successfully ingested streaming data from Random User API.
* Stored raw data in Hadoop HDFS.
* Processed streaming data using Apache Spark.
* Stored processed metrics in InfluxDB.
* Visualized real-time insights using Grafana dashboards.
